Banco de Dados




O que é Banco de Dados?


Um banco de dados é um conjunto organizado de informações estruturadas e relacionadas entre si. É um sistema que permite armazenar, gerenciar e recuperar dados de forma eficiente.
O objetivo principal de um banco de dados é oferecer uma maneira de armazenar grandes volumes de dados de forma organizada, garantindo sua integridade e fornecendo meios para acessar, consultar e atualizar esses dados conforme necessário.
Os bancos de dados são amplamente utilizados em diferentes setores e aplicações, como empresas, instituições governamentais, instituições financeiras e até mesmo em aplicações pessoais. Eles desempenham um papel fundamental no suporte a atividades relacionadas a informações, como registro de transações, gerenciamento de inventário, processamento de pedidos, rastreamento de clientes e muitos outros.
Existem diferentes tipos de bancos de dados, como bancos de dados relacionais, bancos de dados NoSQL, bancos de dados em memória e bancos de dados distribuídos.
Cada tipo possui características e funcionalidades específicas, adequadas para diferentes tipos de aplicações e necessidades.


O que aprendemos?


Iniciamos a matéria de Banco de Dados aprendendo a estrutura do bando, se ela é SQL (Structured Query Language) ou SEQL (Structured English Query Language), tambem vimos o que é um banco de dados e quais os SGBD's (Sistemas de Gerenciamento de Banco de Dados). Em seguida aprendemos sobre a modelagem do BD, aprendendo a como construir o MER (modelo entidade relacionamento) o DER (diagrama entidade relacionamento), aprendemos tambem sobre a cardinalidade que temos que utilizar ao construir o DER, vimos também o modelo logico que é a construção das tabelas que serão implementadas no banco de dados. Após compreender esses conceitos, começamos a aprender o modelo físico, que é a construção do banco de dados. Para isso, aprendemos a linguagem SQL e os comandos DDL (Linguagem de Definição de Dados), DML (Linguagem de Manipulação de Dados), DQL (Linguagem de Consulta de Dados), DTL (Linguagem de Transação de Dados) e DCL (Linguagem de Controle de Dados).



PROJETOS:

Primeiro aprendemos Modelo Entidade Relacionamento (MER), que é uma representação dos requisitos que serão modelados no Banco de Dados. O MER descreve as entidades, os atributos.



Segundo projeto foi o Diagrama Entidade Relacionamento (DER) tem como objetivo traduzir o que foi feito no MER. O DER é utilizado para mapear as entidades, atributos e relacionamentos para facilitar a construção das tabelas, já com suas colunas e chaves primarias e estrangeiras definidas. Ele também auxilia a eliminar redundâncias e garantir a integridade dos dados.
